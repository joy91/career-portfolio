# Data & Feature Design

## Input Data
- Unstructured contract documents (text format)
- Multiple contract types and structures

---

## Key Contract Components Identified
The model was trained to classify and extract:

- Parties involved
- Scope of work
- Payment terms
- Timelines and deadlines
- Intellectual property rights
- Confidentiality clauses
- Dispute resolution clauses
- Digital signature verification

---

## Feature Strategy
- Text preprocessing and normalization
- NLP‑based feature extraction
- Representation suitable for tree‑based models
