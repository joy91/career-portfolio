# Jira-like Workflows

## Standard Delivery Workflow
To Do → In Progress → In Review → Done

## Governance Enhancements
- Validation before moving to Done
- Mandatory fields for key transitions
- Blocked state for dependency issues

---

## Role-based Controls
- Developers: move work within execution states
- Scrum Master: manage sprint scope
- PMO: configure workflows, not day‑to‑day tasks

This ensures **control without micromanagement**.
